# Bitcoin Key Generator from Binary Entropy

A Python application that generates Bitcoin private keys, public keys, and addresses from binary entropy using the secp256k1 elliptic curve.

## Features

- ‚úÖ Convert binary entropy to 256-bit Bitcoin private keys
- ‚úÖ Generate public keys (compressed and uncompressed) using secp256k1
- ‚úÖ Export private keys in WIF (Wallet Import Format)
- ‚úÖ Generate multiple Bitcoin address formats:
  - **P2PKH (Legacy)** - Starts with `1` (mainnet) or `m`/`n` (testnet)
  - **P2WPKH-P2SH (SegWit)** - Starts with `3` (mainnet) or `2` (testnet)
  - **Bech32 (Native SegWit)** - Starts with `bc1` (mainnet) or `tb1` (testnet)
- ‚úÖ Support for both mainnet and testnet
- ‚úÖ Generate random entropy or use custom binary input

## Address Types Explained

### P2PKH (Pay-to-Public-Key-Hash) - Legacy
- **Format**: Base58, starts with `1` (mainnet)
- **Use case**: Original Bitcoin address format
- **Features**: Widely supported, higher transaction fees
- **Example**: `1A1zP1eP5QGefi2DMPTfTL5SLmv7DivfNa`

### P2WPKH-P2SH (Pay-to-Witness-Public-Key-Hash wrapped in P2SH) - SegWit
- **Format**: Base58, starts with `3` (mainnet)
- **Use case**: SegWit addresses compatible with older wallets
- **Features**: Lower fees than P2PKH, backward compatible
- **Example**: `3J98t1WpEZ73CNmYviecrnyiWrnqRhWNLy`

### Bech32 (Native SegWit)
- **Format**: Bech32, starts with `bc1` (mainnet)
- **Use case**: Modern native SegWit addresses
- **Features**: Lowest fees, better error detection, case-insensitive
- **Example**: `bc1qw508d6qejxtdg4y5r3zarvary0c5xw7kv8f3t4`

## Technical Details

### Elliptic Curve
- **Curve**: secp256k1 (same as Bitcoin)
- **Private Key**: 256-bit random number
- **Public Key**: Point on elliptic curve derived from private key

### Key Derivation Process
1. Binary entropy ‚Üí 256-bit integer
2. Validate key is within secp256k1 curve order
3. Generate public key using elliptic curve multiplication
4. Hash public key (SHA-256 + RIPEMD-160)
5. Add version bytes and checksum
6. Encode in Base58 or Bech32

### Dependencies
- **ecdsa**: Elliptic curve cryptography (secp256k1)
- **base58**: Base58 encoding for addresses
- **hashlib**: SHA-256 and RIPEMD-160 hashing (built-in)
